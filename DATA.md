# Dataset setup

There are two paths to a working `data/` tree:

- **Option A: paper bundle.** Download our pre-packaged
  `Paper_Dataset` from Google Drive and unpack into `data/`. This
  is exactly what produced every number in the paper. About 57 GB
  on disk, no preprocessing required. Recommended.
- **Option B: from scratch.** Download ScanNet, 3RScan, ScanScribe,
  and 3DSSG from their official sources and run the
  dataset-creation pipeline yourself. Use this if you cannot get
  bundle access, or if you want to re-run the pipeline.

## Option A: paper bundle from Google Drive

### Access

The bundle is shared on Google Drive on request. Email the
authors for a viewer link. Both ScanNet and 3RScan have their own
terms of use; the bundle's `README.txt` re-states both, and you
should accept them before downloading.

### Drive folder contents

The Drive folder contains three items. Only the first one is
needed for paper reproduction.

| Drive item | Purpose | For paper? |
|---|---|---|
| `Paper_Dataset/` | 3RScan + ScanNet + processed retrieval caches + paper checkpoint. Descriptions are the GPT-4o-mini captions used in the paper (Supp. Sec. 1.4). | Yes. |
| `Full_Dataset_GPT5.5_Descriptor/` | Same scenes, but descriptions regenerated by GPT-5.5 vision for the rebuttal closed-loop-bias defense. | No. Rebuttal artifact only. |
| `Scannet_300_scenes.zip` | 300-scene ScanNet sample for follow-up experiments. | No. Paper Tab. 4(b) uses the 100 scenes listed in [`manifests/scannet_table4_first_100.txt`](manifests/scannet_table4_first_100.txt); Tab. 5 is 3RScan-only. |

### Download

Use a resumable transfer client because the bundle is multi-GB.
`rclone` and `gdown` are common choices:

```bash
# Example with rclone (after configuring a "gdrive" remote)
rclone copy "gdrive:Paper_Dataset" ./data --progress

# Example with gdown (single-folder download)
gdown --folder <Paper_Dataset folder URL>
```

Unpack into `data/` at the repo root.

### Target layout

After unpacking, `data/` should look like this. These paths are
the defaults declared in
[`configs/paths/default.yaml`](configs/paths/default.yaml) and
referenced by every reproducer script, so no edits are needed.

```text
Lang-Loc/
└── data/
    ├── 3RScan/                              # 43 GB, 1319 scenes
    │   └── <scene_uuid>/
    │       ├── labels.instances.annotated.v2.ply
    │       ├── semseg.v2.json
    │       ├── mesh.refined.*               # mesh + segs.v2.json + .mtl + .obj
    │       ├── sequence.zip                 # extracted on first use
    │       └── output/
    │           ├── color/  depth/  pose/    # extracted sequence outputs
    │           ├── camera_pose.json
    │           └── descriptions/            # <frame>.json + <frame>_parsed.json
    ├── scans/                               # 13 GB, ScanNet 100-scene subset
    │   └── <scene_id>/
    │       ├── <scene_id>_vh_clean.ply         # full-res mesh
    │       ├── <scene_id>_vh_clean_2.ply       # decimated mesh
    │       ├── <scene_id>_vh_clean_2.labels.ply
    │       ├── <scene_id>_vh_clean_2.0.010000.segs.json
    │       ├── <scene_id>.aggregation.json
    │       ├── <scene_id>.txt
    │       ├── intrinsic/
    │       └── output/
    │           ├── color/  depth/  pose/    # extracted from .sens
    │           ├── descriptions/            # <frame>.json + <frame>_parsed.json
    │           └── <scene_id>_scene_graph.json
    ├── processed_data/                      # 577 MB, retrieval caches (Tabs 1+2+3)
    │   └── eval_pool/
    │       ├── 3dssg_graphs_518D.pt              # 241 MB, 3DSSG database scene graphs
    │       ├── scanscribe_cleaned_original_518D.pt  # 131 MB, 218-scene distractor pool
    │       ├── scanscribe_graphs_test_518D.pt    # 124 MB, ScanScribe text queries (Tab. 1+2)
    │       ├── db_emb_cache.pt                   # projected DB embeddings
    │       ├── query_emb_cache.pt                # projected text-query embeddings
    │       ├── query_emb_cache_img.pt            # LLM-from-image queries (Tab. 3)
    │       └── db_emb_cache_img.pt               # alias of db_emb_cache.pt for Tab. 3
    └── model_checkpoints/                   # 20 MB, paper checkpoint
        └── graph2graph/
            └── paper/
                └── epoch_70_163_cliprel.pth      # DualSceneAlignerV2 + SimpleGraphMatcher
```

### Sanity check

Run these checks; the expected output is listed in the comment to
the right. Catches the most common unpack failures (wrong target
directory, partial download, missing checkpoint).

```bash
# Top-level directories. If you see numbers smaller than 1319 / 100,
# the bundle is still downloading or the wrong tarball was unpacked.
ls data/3RScan | wc -l                                              # 1319
ls data/scans | wc -l                                               # 100

# Required retrieval caches. Each one must be present; if any are
# missing, Tab. 1+2+3 will fail loudly at load time.
for f in 3dssg_graphs_518D.pt scanscribe_cleaned_original_518D.pt \
         scanscribe_graphs_test_518D.pt \
         db_emb_cache.pt db_emb_cache_img.pt \
         query_emb_cache.pt query_emb_cache_img.pt; do
  test -f "data/processed_data/eval_pool/$f" && echo "OK $f" || echo "MISSING $f"
done

# Paper retrieval checkpoint.
test -f data/model_checkpoints/graph2graph/paper/epoch_70_163_cliprel.pth \
    && echo "OK checkpoint" || echo "MISSING checkpoint"
```

If `ls data/3RScan` shows a single entry named `Paper_Dataset`,
you unpacked one level too deep. Move the contents up by one:

```bash
mv data/Paper_Dataset/* data/ && rmdir data/Paper_Dataset
```

## Option B: from scratch (official sources)

Required if you do not have bundle access, or if you want to
re-run the dataset-creation pipeline (Sec. 3.1) end-to-end.

### ScanNet

1. Sign the [ScanNet Terms of Use](http://www.scan-net.org/). The
   project emails you a personalised `download-scannet.py`.
2. We ship a slightly adapted copy at
   [`tools/download_scannet.py`](tools/download_scannet.py). It
   does not include the access token; add yours.
3. For paper reproduction you only need the 100-scene subset:

   ```bash
   # ~16 GB on disk; the full release is 1.2 TB
   python tools/download_scannet.py --out_dir data --type _vh_clean_2.ply
   python tools/download_scannet.py --out_dir data --type _vh_clean.ply
   python tools/download_scannet.py --out_dir data --type .sens
   # also: .txt, .aggregation.json, _vh_clean_2.0.010000.segs.json
   ```

   The downloader writes to `data/scans/<scene_id>/...`.

### 3RScan

1. Register at the [3RScan project page](http://campar.in.tum.de/public_datasets/3RScan/)
   and accept the licence.
2. Place each scan at `data/3RScan/<scene_uuid>/...` with the
   following files (the pipeline reads them directly):
   - `labels.instances.annotated.v2.ply` (mesh and per-vertex semantic labels)
   - `sequence.zip` (extracted on first use to get RGB frames and poses)
   - `semseg.v2.json` (instance segmentation)

### ScanScribe + 3DSSG

Tabs. 1, 2, and 3 (scene retrieval) need
[3DSSG](https://3dssg.github.io/) (3D Semantic Scene Graph)
annotations and the
[ScanScribe](https://kywind.github.io/scanscribe) caption dataset.
The seven `.pt` files in `data/processed_data/eval_pool/` (listed
in the Option A layout above) are derived from those releases by
[`scripts/retrieval/precompute_eval_embeddings.py`](scripts/retrieval/precompute_eval_embeddings.py).

### Descriptions and parsed graphs

Once raw scans are in place, the per-frame caption JSONs under
`output/descriptions/` are generated by the keyframe-selection and
GPT-4o-mini captioning pipeline. Set `OPENAI_API_KEY` in `.env`
first (see [INSTALL.md](INSTALL.md#environment-variables)). See
[REPRODUCE.md](REPRODUCE.md#dataset-preparation-pipeline) for the
exact commands; expect ~5 to 10 minutes per 100-scene batch on a
single GPU, plus ~$0.01 OpenAI cost per scene.

## Verification

After either option, [`scripts/retrieval/reproduce_paper_tables.sh --skip_precompute`](scripts/retrieval/reproduce_paper_tables.sh)
should complete in under 10 seconds with cache hits. See
[REPRODUCE.md](REPRODUCE.md) for the full reproduction flow.
